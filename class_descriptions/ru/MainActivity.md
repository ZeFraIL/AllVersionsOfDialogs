# 📱 Описание класса: MainActivity

________________________________________
## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность)
*   **Назначение:** Это главная точка входа в приложение. Класс представляет собой "панель управления" или "хаб", содержащий кнопки для запуска различных типов диалоговых окон и уведомлений. Он отвечает за инициализацию пользовательского интерфейса (UI) и реагирование на нажатия кнопок.
*   **Взаимодействие:** Он взаимодействует почти со всеми остальными компонентами проекта:
    *   Создает и отображает экземпляры `AlertDialog` и `MaterialAlertDialogBuilder`.
    *   Запускает подклассы `DialogFragment` (например, `MyDialogFragment`, `FullScreenDialogFragment` и др.).
    *   Открывает другие активности (Activity), такие как `PersistentBottomSheetActivity`, используя **Intent**.

________________________________________
## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `selectedItem` | `int` | Хранит индекс элемента, выбранного в списке с одиночным выбором. | В методах `showSingleChoiceAlertDialog` и `showMaterialSingleChoiceDialog`. |
| `selectedItems` | `ArrayList<Integer>` | Хранит список индексов, выбранных в диалогах с множественным выбором. | В методах `showMultiChoiceAlertDialog` и `showMaterialMultiChoiceDialog`. |
| `progressDialog` | `AlertDialog` | Ссылка на текущее отображаемое окно прогресса. | Используется для программного закрытия диалога после завершения таймера. |

________________________________________
## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Содержит данные о состоянии приложения из предыдущей сессии. |
*   **Что делает метод:**
    1.  Вызывает `super.onCreate` для стандартной настройки активности системой.
    2.  Устанавливает визуальный макет с помощью `setContentView(R.layout.activity_main)`.
    3.  Находит все кнопки по их ID и прикрепляет к ним "слушателей нажатий" (click listeners). При нажатии кнопки выполняется соответствующий метод (например, `showClassicAlertDialog`).
*   **Когда вызывается:** Автоматически системой Android при запуске приложения.
*   **Что важно понимать:** Это начало жизни активности. Если забыть вызвать `setContentView`, экран смартфона будет пустым.

### Имя метода: `showClassicAlertDialog`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает метод:**
    1.  Использует "строитель" `AlertDialog.Builder` для создания всплывающего окна.
    2.  Устанавливает заголовок, сообщение и три кнопки (ОК, Отмена, Позже).
    3.  Вызывает `.show()`, чтобы окно стало видимым пользователю.
*   **Когда вызывается:** При нажатии пользователем на кнопку "Classic Alert Dialog".

### Имя метода: `showProgressDialog`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает метод:**
    1.  Создает диалог Material, который нельзя закрыть простым нажатием назад.
    2.  Запускает `CountDownTimer` на 5 секунд.
    3.  Каждую секунду обновляет текст сообщения, показывая обратный отсчет.
    4.  По окончании таймера закрывает диалог (`dismiss()`).
*   **Когда вызывается:** При нажатии на соответствующую кнопку.

________________________________________
## 4. Жизненный цикл (Activity)
*   **onCreate():** Вызывается при создании активности. Здесь мы настраиваем интерфейс.
*   **onStart(), onResume():** Вызываются, когда активность становится видимой и готовой к работе.
*   **onPause(), onStop():** Вызываются, когда активность уходит на второй план (например, открыто другое приложение).
*   **onDestroy():** Вызывается перед полным уничтожением активности.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** Кнопки (`Button`), текстовые поля (`TextView`), группы переключателей (`RadioGroup`).
*   **Связь с кодом:** Используется `findViewById(R.id.имя_id)` для связи элементов из XML-файла с объектами в Java.
*   **События:** В основном `setOnClickListener` для обработки нажатий.

________________________________________
## 6. Взаимодействие с другими компонентами
*   **Intents:** Используется `new Intent(this, TargetActivity.class)` для перехода на другие экраны.
*   **Fragment Manager:** Используется `getSupportFragmentManager()` для отображения фрагментов-диалогов.

________________________________________
## 7. Общая логика класса
1. Пользователь видит список кнопок.
2. При нажатии на любую кнопку срабатывает метод, который создает объект-"строитель" (Builder).
3. "Строитель" настраивает внешний вид окна (текст, иконки).
4. Вызывается метод `.show()`, и диалог появляется на экране.

________________________________________
## 8. Упрощенное объяснение
Представь, что `MainActivity` — это **Пульт дистанционного управления**.
Каждая кнопка на пульте (интерфейс) связана с определенной функцией (код). Когда ты нажимаешь "Громкость+", пульт посылает сигнал телевизору. Здесь, когда ты нажимаешь "Показать календарь", `MainActivity` посылает сигнал системе Android вывести окно выбора даты. Это "мозг" приложения, который координирует все типы всплывающих окон.
