# 📱 Описание класса: CustomLegacyDialog

________________________________________
## 1. Общая информация
*   **Имя класса:** `CustomLegacyDialog`
*   **Тип:** Обычный класс (наследуется от `Dialog`)
*   **Назначение:** Этот класс демонстрирует "старую школу" или ручной способ создания кастомных всплывающих окон в Android. Он отвечает за показ окна, в котором пользователь может ввести свое имя и подтвердить его.
*   **Взаимодействие:**
    *   Создается и отображается в `MainActivity`.
    *   Использует **Интерфейс обратного вызова** (`OnNameEnteredListener`), чтобы отправить введенное имя обратно в активность.

________________________________________
## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `etName` | `EditText` | Поле ввода, куда пользователь вписывает свое имя. | В `onCreate` для получения текста. |
| `bOk` | `Button` | Кнопка, которую пользователь нажимает для подтверждения. | В `onCreate` для установки слушателя нажатий. |
| `listener` | `OnNameEnteredListener` | Объект, который "ждет" момента, когда пользователь нажмет ОК. | Используется для передачи данных обратно в `MainActivity`. |

________________________________________
## 3. Методы класса

### Имя метода: `CustomLegacyDialog` (Конструктор)
*   **Тип метода:** `public`
*   **Возвращаемое значение:** Нет (это конструктор)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `context` | `Context` | Окружение (обычно `MainActivity`), в котором живет диалог. |
    | `listener` | `OnNameEnteredListener` | Фрагмент кода, который обработает результат ввода. |
*   **Что делает метод:** Инициализирует объект диалога и сохраняет слушателя, чтобы использовать его позже.

### Имя метода: `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void`
*   **Что делает метод:**
    1.  Устанавливает макет из файла `R.layout.custom_legacy_dialog`.
    2.  Убирает стандартную полосу заголовка (`FEATURE_NO_TITLE`).
    3.  Связывает элементы интерфейса (`etName`, `bOk`) с кодом.
    4.  Определяет поведение при нажатии на `bOk`: считывает текст, отправляет его слушателю и закрывает окно.
    5.  Настраивает окно так, чтобы оно было прозрачным и располагалось по центру.
*   **Когда вызывается:** Автоматически при вызове `dialog.show()`.

________________________________________
## 4. Жизненный цикл
*   Это не активность, но у диалога есть свой мини-цикл:
*   **onCreate():** Настройка внешнего вида.
*   **onStart() / onStop():** Моменты появления и исчезновения окна.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `EditText` (ввод текста), `Button` (кнопка действия).
*   **Связь с кодом:** `findViewById` внутри метода `onCreate`.
*   **События:** Обработка нажатия (`setOnClickListener`) на кнопку подтверждения.

________________________________________
## 6. Взаимодействие с другими компонентами
*   **Интерфейс обратного вызова (Callback Interface):** Это самая важная часть. Он определяет "контракт" между диалогом и активностью. Диалог говорит: "Я не знаю, что делать с этим именем, но тот, кто меня использует, должен предоставить функцию `onNameEntered`".

________________________________________
## 7. Общая логика класса
1. `MainActivity` создает диалог и говорит: "Дай мне знать, когда введут имя".
2. Диалог показывает поле для ввода.
3. Пользователь пишет "Иван" и нажимает ОК.
4. Диалог вызывает метод `onNameEntered("Иван")` у слушателя.
5. Диалог закрывается сам по себе (`dismiss`).

________________________________________
## 8. Упрощенное объяснение
Представь, что `CustomLegacyDialog` — это **Посыльный**.
Представь, что ты сидишь в кабинете (`MainActivity`) и тебе нужно узнать имя гостя, но ты занят. Ты посылаешь посыльного (этот диалог). У посыльного есть бланк (интерфейс). Как только гость заполняет бланк и нажимает ОК, посыльный бежит обратно к тебе и отдает листок с именем.
Этот процесс "возврата с ответом" и есть **Слушатель (Listener)**.
