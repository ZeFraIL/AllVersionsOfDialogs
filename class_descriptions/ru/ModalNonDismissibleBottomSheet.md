# 📱 Описание класса: ModalNonDismissibleBottomSheet

________________________________________
## 1. Общая информация
*   **Имя класса:** `ModalNonDismissibleBottomSheet`
*   **Тип:** Фрагмент (конкретно `BottomSheetDialogFragment`)
*   **Назначение:** Этот класс демонстрирует нижнюю панель (Bottom Sheet), которую пользователь **не может** закрыть, потянув её вниз или нажав на пустое место экрана. Это заставляет пользователя взаимодействовать с содержимым или нажать конкретную кнопку "Закрыть".
*   **Взаимодействие:** Запускается из `MainActivity`.

________________________________________
## 3. Методы класса

### Имя метода: `onCreateDialog`
*   **Тип метода:** `public`
*   **Что делает метод:**
    1.  Создает стандартный объект `BottomSheetDialog`.
    2.  Устанавливает `setCancelable(false)`, что запрещает закрытие панели нажатием на фон (затемненную область).
    3.  Добавляет `OnShowListener`: когда панель появляется, метод находит внутренний контейнер `FrameLayout`.
    4.  Получает доступ к `BottomSheetBehavior` и устанавливает `setDraggable(false)`. Это "магическая" строка, которая отключает возможность смахнуть панель пальцем вниз.
*   **Когда вызывается:** При инициализации диалога системой.

### Имя метода: `onCreateView`
*   **Тип метода:** `public`
*   **Что делает метод:** Загружает макет и устанавливает слушатель на кнопку "Close". Поскольку панель нельзя смахнуть, эта кнопка становится **единственным способом** для пользователя закрыть окно.
*   **Когда вызывается:** Для создания визуального содержимого панели.

________________________________________
## 4. Жизненный цикл
*   **onCreateDialog():** Отключение логики "закрыть смахиванием" и "закрыть нажатием вне окна".
*   **onCreateView():** Предоставление пользователю кнопки закрытия.

________________________________________
## 7. Общая логика класса
1. Пользователь нажимает кнопку запуска этой панели.
2. Панель выезжает снизу.
3. Пользователь пытается смахнуть её вниз или нажать на фон, но ничего не происходит.
4. Пользователь "вынужден" прочитать содержимое или совершить действие.
5. Пользователь нажимает на кнопку "Close" внутри панели, чтобы вернуться назад.

________________________________________
## 8. Упрощенное объяснение
Представь, что `ModalNonDismissibleBottomSheet` — это **Обязательное объявление**.
Представь, что в игре всплывает сообщение "Требуется обновление". Вы не должны иметь возможности просто смахнуть его и продолжить игру. Этот класс создает "упрямое" окно, которое будет оставаться на месте до тех пор, пока пользователь явно не подтвердит, что он его увидел, нажав на кнопку.
