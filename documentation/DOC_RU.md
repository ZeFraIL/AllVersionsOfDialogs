## 🎯 Цель проекта
*   **Какую задачу решает приложение?**
    Демонстрирует все популярные и современные способы реализации диалоговых окон, всплывающих меню, нижних карточек (bottom sheets) и уведомлений в Android.
*   **Почему эта задача важна?**
    Диалоги — критически важная часть UX/UI. Разработчикам необходим справочник для выбора правильного типа взаимодействия для конкретных задач.
*   **Целевая аудитория:**
    Android-разработчики, студенты и UI/UX дизайнеры.

## 📌 Требования к приложению
### Функциональные требования
*   Отображение классических и Material 3 диалогов.
*   Реализация диалогов с одиночным и множественным выбором.
*   Создание кастомных макетов (Legacy Dialog и DialogFragment).
*   Демонстрация полноэкранных и прозрачных анимированных диалогов.
*   Интеграция пикеров даты и времени (стандартных и Material).
*   Реализация Bottom Sheets (модальных, постоянных, незакрываемых).
*   Показ Toast, Snackbar и индикаторов прогресса.
### Нефункциональные требования
*   **Производительность:** Мгновенный отклик на действия пользователя.
*   **Удобство использования:** Четкая категоризация типов диалогов.
*   **Надёжность:** Корректная обработка жизненного цикла фрагментов.

## 🧠 Общая архитектура
*   **Выбранный подход:**
    MVC (Model-View-Controller).
*   **Почему выбран именно он:**
    Проект является демонстрационным инструментом. MVC обеспечивает прямой маппинг UI-триггеров (кнопок) на логику контроллера (методы в MainActivity) без излишней сложности.
*   **Основные компоненты системы:**
    *   `MainActivity`: Центральный узел управления.
    *   Подклассы `DialogFragment`: Для модульных диалогов, учитывающих жизненный цикл.
    *   Кастомные диалоги: Для специализированных компонентов интерфейса.

## 🧩 Подробное описание классов
### 📌 Класс: MainActivity
*   **Роль:** Главный контроллер.
*   **Ответственность:** Инициализация интерфейса и обработка нажатий для более чем 30 типов диалогов.
*   **Основные методы:**
    *   `onCreate()` — Настройка макета и слушателей.
    *   `showClassicAlertDialog()` — Демонстрация базовых алертов.
    *   `showMaterialDatePicker()` — Использование компонентов Material Design.
    *   `showProgressDialog()` — Симуляция фоновых задач с обратной связью.
*   **Взаимодействие:** Вызывает различные экземпляры DialogFragment и классы Builder.

### 📌 Класс: CustomLegacyDialog
*   **Роль:** Демонстрация классического класса `Dialog`.
*   **Почему используется:** Чтобы показать, как создавались диалоги до появления DialogFragment.

### 📌 Класс: MyDialogFragment
*   **Роль:** Современная реализация диалога.
*   **Ответственность:** Управление кастомным UI в рамках жизненного цикла фрагмента.

## 🔄 Диаграмма работы приложения
1. Пользователь запускает приложение и видит список кнопок.
2. Пользователь нажимает на кнопку (например, "Material Date Picker").
3. `MainActivity` вызывает соответствующий метод.
4. Система отрисовывает диалог поверх текущего экрана.
5. Пользователь взаимодействует с диалогом, результат отображается через Toast.

## 🎨 UI/UX анализ
*   **Почему интерфейс сделан именно так:** Интерфейс ориентирован на кнопки для быстрого доступа к любому типу диалога.
*   **Принципы:**
    *   **Простота:** Одно действие на одну кнопку.
    *   **Логичность:** Связанные диалоги сгруппированы.
    *   **Доступность:** Высокий контраст и стандартные компоненты Material.
*   **Что можно улучшить:** Добавить строку поиска или категории (вкладки) для еще лучшей навигации.

## ⚙️ Работа с потоками
*   **Используются:**
    *   `Handler` / `Looper.getMainLooper()`
    *   `CountDownTimer`
    *   `Thread` (для симуляции прогресса)
*   **Почему:** Для имитации фоновой работы и безопасного обновления UI.
*   **Предотвращение проблем:**
    *   **ANR:** Длительные задачи выносятся в фоновые потоки.
    *   **Утечки:** Своевременное закрытие диалогов в `onFinish` или `onDestroy`.
